🐟 Piranhas KI

Ein Cython-optimierter Game-Bot für die Software-Challenge — vollständig erklärt

🎯 Überblick

Was ist das hier?

Dies ist ein Game-Bot für das Spiel "Piranhas" der Software-Challenge Germany. Der Bot verwendet Künstliche Intelligenz in Form eines Minimax-Algorithmus mit Alpha-Beta-Pruning, um den bestmöglichen Zug zu berechnen.

Der Code ist in Cython geschrieben — einer Mischung aus Python und C — um maximale Performance zu erreichen (10-100× schneller als reines Python).

Architektur des Bots

📋 board.pyx

Spielfeld-Repräsentation, Zugvalidierung und Zuggenerierung. Speichert das 10×10-Brett als kompaktes Array.

#️⃣ zobrist.pyx

Zobrist-Hashing für schnelle Positionserkennung. Ermöglicht effiziente Transpositionstabellen.

⚖️ evaluate.pyx

Positionsbewertung mit Schwarm-Analyse. Berechnet, wie gut eine Stellung für ein Team ist.

🔍 search.pyx

Alpha-Beta-Suche mit Iterative Deepening. Das "Gehirn" des Bots, das Züge vorausberechnet.

🎮 client_cython.py

Verbindung zum Spielserver. Empfängt Spielstände und sendet berechnete Züge zurück.

⚙️ setup.py

Build-Konfiguration. Kompiliert Cython-Code zu schnellen C-Erweiterungen.

Der Spielablauf

Ablauf jedes Zuges:
  1. Server sendet aktuellen GameState
  2. from_game_state() konvertiert zu internem Board
  3. iterative_deepening() sucht den besten Zug
  4. Bester Zug wird an Server zurückgesendet
💡 Kernkonzept: Der Bot denkt mehrere Züge voraus und bewertet alle möglichen Spielverläufe. Er wählt den Zug, der zum besten Ergebnis führt — auch wenn der Gegner optimal spielt.

⚡ Cython Grundlagen

Was ist Cython?

Cython ist eine Programmiersprache, die Python-Syntax mit C-Datentypen kombiniert. Der Cython-Compiler übersetzt den Code zu C, das dann zu nativen Maschinencode kompiliert wird.

Resultat: 10-100× schneller als reines Python, aber fast genauso einfach zu schreiben.

Compiler-Direktiven

# cython: language_level=3, boundscheck=False, wraparound=False, cdivision=True

wraparound=FalseKeine negativen Indizes (wie arr[-1])⚡ Schnellerer Array-Zugriff

⚠️ Achtung: Diese Optimierungen sind unsicher! Bei boundscheck=False kann ein falscher Index zu Speicherfehlern führen. Nur verwenden, wenn der Code korrekt ist!

📋 Spielfeld (board.pyx)

Bit-Packing: Speichereffiziente Felder

Jedes Spielfeld speichert zwei Informationen: das Team (0-3) und den Wert des Fisches (0-4). Statt zwei Variablen zu verwenden, werden beide in einem einzigen Byte kombiniert.

■ Bits 0-1: Team (0-3)    ■ Bits 2-4: Wert (0-7) 

# Team-Konstanten
DEF TEAM_NONE = 0   # Leer
DEF TEAM_ONE = 1    # Team 1
DEF TEAM_TWO = 2    # Team 2
DEF TEAM_SQUID = 3  # Krake (Hindernis)

# Feld erstellen: Team in Bits 0-1, Wert in Bits 2-4
cdef int8 make_field(int team, int value) noexcept nogil:
    return <int8>((team & 0x3) | ((value & 0x7) << 2))
    #              ↑ Bits 0-1     ↑ Bits 2-4 (nach links geschoben)

# Team extrahieren (Bits 0-1 maskieren)
cdef int get_team(int8 field) noexcept nogil:
    return field & 0x3  # 0x3 = 0b00000011

# Wert extrahieren (nach rechts schieben, dann maskieren)
cdef int get_value(int8 field) noexcept nogil:
    return (field >> 2) & 0x7  # 0x7 = 0b00000111
💡 Beispiel: Team 1 mit Wert 4
make_field(1, 4) = (1 & 0x3) | ((4 & 0x7) << 2) = 1 | (4 << 2) = 1 | 16 = 17 = 0b00010001

Das Spielfeld als 1D-Array

Das 10×10 Spielfeld wird als eindimensionales Array mit 100 Elementen gespeichert. Die Umrechnung von 2D zu 1D: index = x * 10 + y

cdef class CBoard:
    cdef int8[100] fields   # Flaches Array für 10×10 Brett
    cdef public int turn   # Aktueller Zug (0-59)

    # 2D-Koordinaten → 1D-Index umrechnen
    cdef int8 get_field(self, int x, int y) noexcept nogil:
        return self.fields[x * 10 + y]
        #                   ↑ Spalte × Breite + Zeile

Zugweiten-Berechnung

Bei Piranhas hängt die Zugweite von der Anzahl der Fische auf der Bewegungslinie ab:

# Zählt Fische (beide Teams) auf einer Linie
cdef int count_fish_on_line(CBoard board, int x, int y, int dx, int dy) noexcept nogil:
    cdef int count = 0
    cdef int nx = x + dx   # Startposition überspringen
    cdef int ny = y + dy
    
    while 0 <= nx < 10 and 0 <= ny < 10:  # Solange im Brett
        field = board.get_field(nx, ny)
        if get_team(field) == TEAM_ONE or get_team(field) == TEAM_TWO:
            count += 1
        nx += dx
        ny += dy
    
    return count
🎮 Spielregel: Ein Fisch bewegt sich so viele Felder, wie Fische (beider Teams!) auf seiner Bewegungslinie stehen — inklusive sich selbst.

#️⃣ Zobrist Hashing

Das Problem: Gleiche Stellungen erkennen

Bei der Spielbaum-Suche erreichen wir oft dieselbe Stellung über verschiedene Zugfolgen. Ohne Erkennung würden wir diese Stellungen mehrfach analysieren — Zeitverschwendung!

Lösung: Jede Stellung bekommt einen eindeutigen Hash-Wert. Gleiche Stellung = gleicher Hash.

Die Idee hinter Zobrist Hashing

1
Zufallszahlen vorberechnen

Für jede Kombination aus (Position, Team, Wert) eine zufällige 64-Bit-Zahl generieren.

2
Hash berechnen per XOR

Den Hash der Stellung durch XOR aller relevanten Zufallszahlen berechnen.

3
Inkrementelle Updates

Bei einem Zug nur die Änderungen XOR-en — viel schneller als neu berechnen!

Die Zufallszahlen-Tabelle

# 4D-Array: [x][y][team][value] → Zufallszahl
cdef uint64[10][10][4][5] ZOBRIST_PIECE
# Für jeden Zug eine Zahl (um Zugwiederholungen zu erkennen)
cdef uint64[61] ZOBRIST_TURN

cpdef void init_zobrist():
    srand(42)  # Deterministischer Seed für Reproduzierbarkeit
    
    for x in range(10):
        for y in range(10):
            for t in range(4):
                for v in range(5):
                    ZOBRIST_PIECE[x][y][t][v] = _rand64()

Inkrementelles Update

cpdef uint64 update_hash_move(uint64 old_hash, ...):
    h = old_hash
    
    # Zug-Hash ändern
    h ^= ZOBRIST_TURN[old_turn]   # Alten Zug raus-XORen
    h ^= ZOBRIST_TURN[new_turn]   # Neuen Zug rein-XORen
    
    # Figur bewegen
    h ^= ZOBRIST_PIECE[start_x][start_y][team][value]   # Von Start entfernen
    h ^= ZOBRIST_PIECE[target_x][target_y][team][value] # Zu Ziel hinzufügen
    
    return h
🔑 XOR-Eigenschaft: A ^ B ^ B = A
Wenn wir etwas zweimal XOR-en, verschwindet es wieder!

⚖️ Bewertungsfunktion

Wie gut ist eine Stellung?

Die Bewertungsfunktion gibt jeder Spielstellung einen numerischen Score. Positiv = gut für uns, negativ = gut für den Gegner.

Bei Piranhas gewinnt das Team mit dem wertvollsten Schwarm — einer zusammenhängenden Gruppe von Fischen.

Schwarm-Analyse mit BFS

# Datenstruktur für Schwarm-Ergebnisse
cdef struct SwarmData:

Die Bewertungsformel

cpdef double evaluate(CBoard board, int our_team):
    our_data = compute_swarm_data(board, our_team)
    opp_data = compute_swarm_data(board, opp_team)
    
    value = 0.0

    value += (opp_data.num_swarms - 1) * 4.0
    
    # 3. Material + 4. Isolierte Fische + 5. Distanz
    ...
    
    return value

🔍 Suchalgorithmus

Minimax mit Alpha-Beta-Pruning

Der Suchalgorithmus simuliert das Spiel mehrere Züge voraus. Er geht davon aus, dass beide Spieler optimal spielen — wir maximieren, der Gegner minimiert.

Der Spielbaum

Wir (MAX)
Score: 5
Gegner (MIN)
Score: 5
Gegner (MIN)
✂️ Cutoff
Gegner (MIN)
Score: 3

Alpha-Beta erklärt

α
Alpha = bester Score, den WIR garantiert erreichen
β
Beta = bester Score, den der GEGNER garantiert erreicht

Wenn β ≤ α, wird der Zweig abgeschnitten — der Gegner würde ihn nie zulassen!
